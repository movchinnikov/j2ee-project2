Overview

Project Overview

A residential cleaning service platform built as a distributed microservice system using Spring Boot. Clients book cleaning sessions, cleaners get auto-assigned via round-robin, and every state change triggers a guaranteed Kafka event via the Transactional Outbox Pattern.

1
Client registers & books
Selects property, service type, date → price calculated in real-time
2
Order paid → cleaner auto-assigned
Round-robin algorithm picks available cleaner from pool
3
Events published to Kafka
All state transitions are durable events — at-least-once delivery

Messaging

Kafka + Transactional Outbox Pattern

Why Outbox?

Without it, a service could commit the DB but crash before sending to Kafka — losing the event forever. With Outbox, both writes happen in one atomic transaction.

1
Atomic write (same DB transaction)
Business record + outbox_events row committed together
2
Relay polls every 5 seconds
SELECT WHERE status=PENDING, publish to Kafka, mark PROCESSED
3
Consumer reacts idempotently
Order Service: auto-creates CleanerProfile on user/role events

// 1. Business TX — ONE transaction
@Transactional
public AuthResponse register(...) {
  UserEntity user = userRepo.save(newUser);

  outboxPublisher.publish(
    "User", userId,
    "user.registered",
    UserRegisteredEvent.builder()
      .userId(user.getId())
      .roles(roles)
      .build()
  ); // ← same TX as user save
  return tokens;
}

// 2. Relay — every 5 sec (async)
@Scheduled(fixedDelay = 5000)
void relay() {
  outboxRepo.findPendingBatch()
    .forEach(e -> kafkaTemplate.send(
      e.getEventType(),
      e.getAggregateId(),
      e.getPayload()
    ));
}
At-Least-Once Delivery
FAILED events are retried on next poll cycle. Consumers are idempotent.

Docker Build Strategy
# Multi-stage build (iac-service & order-service)
FROM maven:3.9.6-eclipse-temurin-17 AS builder

# Step 1 — install shared-lib to local Maven repo
COPY shared-lib/pom.xml shared-lib/pom.xml
COPY shared-lib/src     shared-lib/src
RUN  mvn -f shared-lib/pom.xml clean install -q

# Step 2 — cache service deps
COPY iac-service/pom.xml iac-service/pom.xml
RUN  mvn -f iac-service/pom.xml dependency:go-offline -q

# Step 3 — build service JAR
COPY iac-service/src iac-service/src
RUN  mvn -f iac-service/pom.xml clean package -DskipTests -q

FROM eclipse-temurin:17-jre AS runtime
COPY --from=builder /build/iac-service/target/*.jar app.jar
